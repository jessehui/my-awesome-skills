# fork-context skill — 设计文档

日期：2026-06-25
状态：已与用户对齐，待实现

## 目标

在一个很长的当前会话里（例：正在做某平台性能调优，反复尝试过多种思路），
用户突然想到一个**与当前事相关、但分叉出去的新话题**（例："内存分配器会不会有关？"），
希望在一个**相对干净的上下文**里聊这个分叉，而**不污染当前主线会话**。

`fork-context` skill 做的事：针对该分叉话题，从当前会话萃取相关上下文，写成一份
**自包含 brief 文件**，并产出一段可直接粘进新 `claude` 会话的 **paste-prompt**。

## 核心约束：保护主线会话上下文干净

这是本设计最关键的一点。skill 在当前会话执行，若直接"回扫对话→生成 brief→打印全文"，
会把整段分叉的展开灌回当前上下文——与目标相反。

**解决办法：用 `fork` 类型 subagent 做隔离萃取。** `fork` 有两个契合性质：

1. 继承当前完整对话上下文 —— 读得到这一长串调优记录，能做萃取；
2. 在后台独立上下文运行，工具输出不回灌主会话 —— 回扫、组织 brief、写文件等"重活"
   全发生在 fork 自己的上下文里，不进入当前 session。

主线会话的上下文**净增量**因此只有：用户那条调用命令 + fork 返回的最小确认（路径 +
3 行 paste-prompt）。分叉话题的所有展开被隔离在 fork 里、随 fork 结束而丢弃。

### 不可消除的残留（已与用户确认接受）

- 用户的调用命令本身（含分叉问题原文）必然留在主会话——这是用户在主会话敲入的，无法避免，无害。
- paste-prompt 采用 **(b) 方案**：主会话打印 3 行方便用户直接复制，接受这几行残留，
  换取"无需 `cat` 取文件"的便利。

## 执行流程

```
主会话:  /fork-context <分叉问题>
   │
   ├─ 主会话唯一动作: 立即 dispatch 一个 fork subagent,
   │                  把萃取指令 + 分叉问题原文传给它。
   │                  主会话自身不做回扫/萃取/brief 生成。
   │
   └─ fork (继承上下文, 隔离运行):
        1. 回扫当前对话, 按分叉问题做相关性过滤 (只抽相关内容, 非整段转储)
        2. 组织成 brief 的 5 个固定段落
        3. 写 brief 文件到当前工作目录
        4. 组织 paste-prompt
        5. 最小化返回: 仅 brief 文件绝对路径 + 3 行 paste-prompt
              (绝不把 brief 全文返回主会话)
   │
   └─ 主会话收到并打印: 一行路径确认 + 3 行 paste-prompt (供用户复制)
```

## 调用方式

```
/fork-context 内存分配器会不会和这个性能问题有关
```

- 分叉问题作为参数传入。
- 无参数时，先反问"你想分叉到哪个话题？"，拿到后再继续。

## brief 文件

- 落地目录：当前工作目录。
- 文件名：`fork-context-YYYY-MM-DD-<话题slug>.md`（时间戳 + slug，避免多次分叉互相覆盖）。
- 固定 5 段：

| 段 | 内容 |
|---|---|
| 背景与目标 | 在做什么、目标、环境/硬件/软件栈等关键设定。让新 session 知道"场景"。 |
| 已试过的方案与结果 | 之前反复尝试过哪些思路、各自结果、哪些已被证伪/排除。避免新 session 重踩老路。 |
| 关键数据与结论 | 关键数值/指标/profiling 结果、已经立足的结论。新 session 可直接引用。 |
| 与新话题的连接点 | 明确点出"为什么会想到这个分叉"——当前哪个现象/停滞点让用户怀疑到新方向。 |
| 来源溯源 | brief 提炼自哪些文件/路径/上下文，方便新 session 回溯原始记录。 |

## 萃取准则（skill 的核心价值，写在 SKILL.md 里、由 fork 执行）

- **相关性过滤**：只抽与分叉话题相关的内容，不是整段对话转储。
- **自包含**：任何代称（"它""那个方案""之前那次"）都要展开成新会话能看懂的具体名词、
  路径、数值。新 session 零背景，brief 必须独立可读。
- **保留已排除项**：明确写出哪些思路已被试过且无效，防止新 session 重复劳动。
- **诚实标注来源**：溯源段如实列出 brief 依据的文件/路径/上下文位置。

## paste-prompt 形态

主会话打印、用户复制到新 `claude` 会话。约 3 行，内嵌分叉问题原文 + brief 文件绝对路径：

```
我从另一个会话分叉出一个新话题。背景见 <绝对路径>/fork-context-2026-06-25-malloc.md，
请先读它。我的问题是：内存分配器会不会和这个性能问题有关？
```

- 内嵌用户的分叉问题**原文**。
- 给出 brief 文件的**绝对路径**（用户在任何目录新开会话都能让它读到）。

## skill 形态

- 纯流程 skill，单个 `SKILL.md`，无脚本、无外部依赖。
- 位置：`my-skills/fork-context/`（与同级 `verify` skill 并列）。
- frontmatter `description` 需触发于：用户在长会话里想把一个相关但分叉的话题
  拎到干净上下文中讨论；关键词如"分叉""另开一个干净的会话聊""fork 一下这个话题"。

## 显式排除（YAGNI）

- 不让 subagent 替用户把新话题"聊掉"——产物只是 brief + prompt，新话题仍由用户在
  新会话亲自展开。
- 不做自动新开 session 的尝试（skill 无法真正开顶层 session）。
- 无脚本、无配置、无状态持久化。
